# Publishable memory opportunities

LinkedIn Studio should start from what the company's memory makes publishable, not from a LinkedIn template. A publishable memory opportunity is a piece of content that can be created from existing memories because the memories contain a pattern, moment, artifact, learning, tension, or phrase worth making public.

Memory Store may contain context from Slack, shipped work, customer calls, support, docs, founder notes, edits, prior posts, and performance reports. Treat these as memory sources, not tool assumptions. Recall through Memory Store and work only from the memories returned.

## Opportunity types

Use these types to classify what recall found. A candidate can match more than one, but pick the primary type before drafting.

| Type | Use when memory reveals... | Usually becomes... |
| --- | --- | --- |
| `customer_pattern` | three or more customers share the same pain, workaround, ask, or misconception | user insight, category reframe |
| `customer_language` | a customer phrase, objection, or description says the category better than the company does | customer story, objection, founder POV |
| `artifact_to_public` | an internal artifact is useful outside the company: report, dashboard, PR, memo, teardown, checklist, usage log | practical post, teardown, lessons learned |
| `shipped_trace` | a shipped feature or fix has a clear before, cause, and user-facing change | shipped work, mini case study |
| `decision_archaeology` | memory shows how a product or GTM decision evolved through tradeoffs and evidence | founder POV, build-in-public, lessons learned |
| `internal_debate` | the team debated a category belief, product choice, or customer tradeoff that can be safely public | contrarian POV, category argument |
| `support_theme` | repeated support issues expose a user mental model or product gap | user insight, practical guide |
| `sales_objection` | a reasonable objection keeps appearing in calls or emails | objection post, trust-building post |
| `founder_belief` | the author or company has a defensible view backed by memory | founder POV, manifesto fragment |
| `team_learning` | the team changed how it works because of a concrete lesson | retrospective, build-in-public |
| `before_after` | memory shows a clear transformation over time | customer story, shipped trace, operational lesson |
| `performance_learning` | prior posts, edits, approvals, or metrics show what worked for this brand or author | format choice, hook choice, CTA choice |
| `manifesto_fragment` | memory contains a belief, phrase, or atmospheric line that sets the world the company wants to build | short philosophical post, brand POV |

## Opportunity card

Before drafting, write a compact card for each strong candidate:

```text
Opportunity: <one sentence>
Type: <primary opportunity type>
Source memories: <memory IDs>
Audience: <specific ICP reader>
Why now: <recency, launch, repeated signal, or strategic moment>
Tension: <belief, tradeoff, mistake, contradiction, or emotional stakes>
Proof: <quote, artifact, shipped change, repeated pattern, metric, or decision trail>
Brand fit: <pillar, emerging signal, or reason to test>
Voice fit: <author rule, phrase, rhythm, or constraint>
Prior learning: <what previous posts/edits/performance suggest, if recalled>
CTA job: <save, reply with a story, DM, try a workflow, challenge the view>
```

## Angle score

Score each candidate 0-5 on six dimensions. Draft the highest-scoring opportunities first.

- **Audience pain** — a specific reader feels this, not "people in business."
- **Novelty** — the angle is not common advice with new nouns.
- **Proof** — a memory supplies concrete evidence: quote, artifact, pattern, number, shipped trace, or decision trail.
- **Tension** — the post has a belief, tradeoff, mistake, reversal, disagreement, or emotional stake.
- **Usefulness** — the reader can save, forward, apply, or argue with it.
- **Author credibility** — this author or company has standing to say it.

Guidance:

- 26-30: publish candidate.
- 22-25: draft if voice and proof are strong.
- 18-21: hold unless the user explicitly wants it.
- Under 18: drop or ask for more source material.

Do not let a weak score become a generic post. A weak opportunity is usually a recall problem, not a writing problem.

## Shapers after opportunity selection

The opportunity is the source. These shape it:

- **Brand** — what the company can credibly say, which pillar it serves, what is approved or taboo.
- **Author voice** — how this person writes, what they refuse, and which claims they can own.
- **Prior performance** — what hooks, formats, CTAs, lengths, and topics have worked or failed before.
- **Channel craft** — hook, line breaks, pacing, proof placement, and CTA for LinkedIn.
- **CTA** — the action or response the post is designed to invite.

Do not start from the CTA or the format. Start from the opportunity, then choose the shape that serves it.

## CTA jobs

Pick one CTA job per draft:

- **Story reply** — ask readers for their version of the same pattern.
- **Save** — name the practical use of the post.
- **Disagreement** — invite a specific counterargument, not "thoughts?"
- **Diagnostic** — ask readers to inspect a workflow, metric, or decision.
- **DM / inbound** — use only when the post has already delivered value.
- **Try it** — for shipped work when the user can act immediately.
- **No CTA** — valid for manifesto fragments or posts that should land as a statement.

The CTA should emerge from the opportunity. A support theme usually wants a diagnostic CTA. A customer pattern usually wants a story reply. A practical artifact usually wants a save. A founder belief usually wants disagreement.
