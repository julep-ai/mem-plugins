# Support Community Memory Teams

Status: proposed expanded ICP
Target first batch: 15 accounts after shadow sample
Primary learning intent: support and community memory fit

## Persona

Devrel lead, community lead, support lead, developer advocate, open-source maintainer, support/community founder.

## Pain

Repeated questions, docs gaps, user issues, onboarding context, and community history vanish between Slack, Discord, GitHub, docs, support tools, and agents.

## Hard Criteria

- Active support, devrel, open-source, or community surface.
- Public repeated-question, docs, onboarding, issue, or community-growth signal.
- Buyer or owner can be identified.

## Trigger Signals

- Active Discord, Slack, GitHub issues, forum, docs, or public support channel.
- Public docs churn, repeated support questions, community growth, onboarding complaints, or support hiring.
- Hiring support, community, devrel, docs, or developer education roles.

## Proof Path

Show a support/community context brief with repeated questions, docs gaps, owner context, reusable answers, and follow-up loops.

## Exclusions

- Communities with no business buyer.
- Static open-source repos without active support load.
- Accounts where the support surface is private and cannot be verified.

## Websets Query

Developer tools, AI products, open-source projects, or B2B SaaS teams with active Discord, Slack, GitHub issues, docs, support, or community workflows where repeated answers and user context need to survive across agents.
