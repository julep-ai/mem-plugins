# Coding Agent Continuity Teams

Status: proposed
Target first batch: 20 accounts after shadow sample
Primary learning intent: technical-founder and engineering-lead resonance

## Persona

Technical founder, engineering lead, AI platform lead, devtools/community lead, MCP/plugin builder, or internal developer-platform owner.

## Pain

Coding agents repeatedly rediscover repo context, constraints, failed attempts, conventions, and decisions across Claude, Codex, Cursor, GitHub, and local project state.

## Hard Criteria

- Public agent/devtools/MCP/coding workflow signal.
- Active engineering product or team.
- Technical buyer can be identified.
- Live source URL exists.

## Trigger Signals

- Public MCP server, agent framework, coding-agent product, or AI developer-tooling launch.
- GitHub activity around agents, code generation, internal developer platforms, or automation.
- Docs/changelog velocity suggesting active agent/product work.
- Public posts about Claude Code, Codex, Cursor, repo agents, or context loss.

## Proof Path

Show record in one agent and recall in another, or a repo/project continuity readout that preserves decisions, attempts, constraints, and next actions.

## Exclusions

- Generic AI startups with no public agent/devtools workflow.
- Teams with no clear technical buyer.
- Accounts where the only evidence is old funding or a broad AI category.

## Websets Query

Engineering-heavy startups, devtools companies, MCP/plugin builders, coding-agent products, and AI platform teams using Claude Code, Codex, Cursor, agents, or MCP where project context and decisions must persist across sessions.
