# Vertical AI App Builders

Status: proposed expanded ICP
Target first batch: 10 accounts after shadow sample
Primary learning intent: product-embedded memory wedge

## Persona

Vertical AI founder, product lead, applied AI engineer, or AI app builder.

## Pain

Their own users or customer teams need context to persist across sessions, tasks, documents, and agents, but the product risks becoming another stateless chatbot or generic RAG surface.

## Hard Criteria

- Building an AI product in a vertical workflow.
- User or team memory is plausibly part of the product experience.
- Public signal around personalization, user history, long-running workflows, agents, or memory.

## Trigger Signals

- Launching AI products in legal, healthcare, finance, real estate, procurement, recruiting, education, customer ops, or similar verticals.
- Public docs/posts about memory, personalization, agents, user history, long-running cases, client context, or workflow continuity.
- Hiring applied AI, product, solutions, or customer implementation roles.

## Proof Path

Show a scoped memory profile or MCP-backed context loop that records and recalls user/team context across sessions.

## Exclusions

- Wrapper apps with no durable workflow.
- One-shot generation tools.
- Products where memory would create unclear privacy or scope issues without an obvious control model.

## Websets Query

Vertical AI app builders in legal, healthcare, finance, real estate, procurement, recruiting, education, or customer operations where user or team memory, personalization, long-running workflows, or agent context persistence is part of the product experience.
