## Campaign Hypothesis

AI-heavy teams are adding agents faster than they are adding shared execution context. The pain is not "we need a better knowledge base"; it is that Claude, Codex, Cursor, Slack, meetings, docs, and email all know different parts of the work, so the next agent starts without the decisions, obligations, taste constraints, customer context, and open loops that still matter.

Memory Store should sell the operating layer: one memory for the team's agents, built from meetings, Claude sessions, Slack, and other work surfaces, available wherever the team and agents already work.
